LiveOrder v3 기획서

카카오톡 챗봇 기반 주문 시스템 | 작성: Eddy PM | 2026-04-09

핵심 컨셉: 고객이 카카오톡에서 채팅으로 상품을 주문하고, 봇이 주문 접수부터 배송 조회까지 관리하는 시스템.

1. 프로젝트 개요

항목내용
프로젝트명LiveOrder v3
버전 히스토리v1: 웹 기반 e-커머스 (종료) → v2: DROP → v3: 카카오톡 챗봇
핵심 기술카카오 오픈빌더 + Node.js 스킬 서버 + Ollama/Gemma4 + PostgreSQL
결제PortOne (카카오페이 포함)
호스팅Coolify (자체 서버)

2. 이해관계자

역할설명할 일
플랫폼 (우리)시스템 개발/운영오픈빌더 봇 관리, 스킬 서버, DB, 인프라 전체
판매자상품 등록/주문 관리관리자 페이지에서 상품 등록, 주문 확인, 배송 처리
고객상품 구매카카오톡에서 채팅으로 주문

3. 시스템 아키텍처

고객 (카카오톡) │ liveorder 채널 친구추가 ▼ 카카오 봇 엔진 (오픈빌더) │ HTTP POST (스킬 요청) ▼ ┌─────────────────────────────────────┐ │ 스킬 서버 (Node.js) │ │ │ │ 1. 즉시 응답 (useCallback: true) │ │ 2. 비동기 처리: │ │ ├─ 판매자/상품 DB 조회 │ │ ├─ LLM 대화 처리 (Gemma4) │ │ └─ 주문/결제 관리 │ │ 3. callbackUrl로 최종 답변 전송 │ └─────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────┐ │ PostgreSQL │ Ollama/Gemma4 │ │ (판매자/상품/ │ (대화 AI) │ │ 주문/배송) │ localhost:11434 │ └─────────────────────────────────────┘
카카오 오픈빌더 제약: 봇은 자기 채널에만 연결 가능. 판매자 채널에 우리 봇을 넣을 수 없음.
→ 우리 채널(liveorder) 1개에서 판매자를 선택하는 구조로 운영.

4. 고객 주문 플로우

단계고객 행동봇 응답UI 타입
1liveorder 채널 친구추가환영 메시지 + 안내simpleText
2"주문하기" 입력판매자 목록 표시listCard
3판매자 선택해당 판매자 상품 목록carousel (commerceCard)
4상품 선택옵션 선택 (사이즈/색상 등)quickReplies
5옵션 확정수량 확인quickReplies
6수량 입력배송지 입력 요청simpleText
7배송지 입력 (대화)주문 요약 + 결제 버튼basicCard + webLink 버튼
8결제 버튼 클릭외부 결제 페이지 (PortOne)webLinkUrl
9결제 완료주문 확인 알림톡알림톡
10"배송 조회" 입력택배 배송 상태simpleText

5. 판매자 관리 기능

5-1. 판매자 온보딩

  • 관리자 페이지에서 판매자 등록 (사업자명, 연락처, 카테고리)
  • 상품 등록 (이름, 가격, 옵션, 이미지, 재고)
  • 판매자별 정산 정보 설정

5-2. 판매자 관리자 페이지

기능설명
상품 관리상품 등록/수정/삭제, 옵션 관리, 재고 관리
주문 관리주문 목록 확인, 상태 변경 (접수→배송준비→배송중→완료)
배송 관리운송장 등록, 배송 상태 업데이트
매출 현황일별/월별 매출, 주문 통계

6. DB 스키마 (핵심 테이블)

테이블주요 필드설명

productsid, seller_id, name, price, image_url, status상품

chat_sessionsid, kakao_user_id, seller_id, state, context대화 세션 (LLM 히스토리)
